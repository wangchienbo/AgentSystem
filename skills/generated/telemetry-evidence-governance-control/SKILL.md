# Telemetry Evidence Governance Control

Structural subordinate skill for the `telemetry-evidence-governance` domain in AgentSystem.

## Scope
Owns focused work related to telemetry, upgrade/evidence logging, refinement and risk governance, operator dashboards, and review/observability surfaces.

## Entry conditions
Use this skill when the requested work is primarily inside telemetry, evidence, governance, or operator-facing observability domains.
Escalate back to `agentsystem-master-control` when changes substantially impact broader platform-core boundaries or cross-domain architecture.

## Owned focus
- telemetry / upgrade-evidence / governance read-model surfaces
- operator dashboard and review-related docs and service flows
- risk/refinement governance flows where governance responsibility is primary

## Operating rules
- prefer governance and telemetry context first
- escalate when changes materially affect platform-core or runtime/control-plane contracts
- update project control artifacts when governance interfaces or dependencies materially shift
