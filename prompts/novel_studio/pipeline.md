# 流水线写作技能（Pipeline）

## 核心能力
- 从大纲生成完整章节（规划→场景拆分→逐段写作→写入章节存储）
- 批量生成多章节
- 带角色对话和世界场景描写的完整章节

## 操作方式
Pipeline 是 Novel Studio 内置的自动化流水线，它按以下步骤工作：
1. **规划（Plan）**：获取小说大纲，分析当前章节规划，确定本章节要写的内容
2. **场景拆分（Scene Breakdown）**：将章节拆分成多个场景，每个场景设定 POV 角色、地点、目标
3. **逐段叙事（Narrative）**：逐个场景生成叙事内容，含角色对话、动作描写、环境氛围
4. **记忆更新（Memory）**：将本章节的关键事件记录到角色记忆中

调用 `call_asset_method(asset:novel_studio:v1, write_chapter, {"novel_id": "xxx"})` 触发流水线。

写完后需调用 `get_novel` 确认章节已生成。

## 使用时机
- 用户说"写一章"、"生成下一章"、"帮我写"——调用流水线
- 用户说"续写"——调用流水线
- 用户说"写接下来的3章"——建议先调流水线写一章，确认后再继续

## 写作原则
1. **忠于大纲**：严格遵循大纲中的章节规划
2. **POV 一致性**：每章指定 POV 角色，保持视角统一
3. **场景完整性**：每个场景有开始、发展、结尾
4. **悬念驱动**：章节末尾留钩子
