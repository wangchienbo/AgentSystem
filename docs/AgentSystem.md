# AgentSystem

## 概述

AgentSystem 是一个智能交互引擎系统，专为多轮工具调用和任务执行而设计。它提供了一个统一的框架来管理各种系统资产、执行工具调用，并实现智能的任务编排。

## 核心功能

### 1. 工具调用系统

AgentSystem 提供了一套完整的工具集：

| 工具名称 | 功能描述 |
|---------|---------|
| `exec_shell` | 执行系统命令和脚本 |
| `read_file` | 读取文件内容 |
| `write_file` | 写入或覆盖文件内容 |
| `edit_file` | 精确编辑文件（搜索替换） |
| `list_files` | 列出目录内容 |
| `search_files` | 搜索文件内容 |
| `find_tool` | 搜索可用工具 |
| `call_asset_method` | 调用运行时资产方法 |
| `ask_clarification` | 向用户询问更多信息 |
| `unclear` | 当无法理解用户意图时进行引导 |

### 2. 资产管理系统

- **统一调用接口**: 所有系统资产通过 `call_asset_method` 统一调用
- **动态资产发现**: 支持运行时资产的动态发现和管理
- **多资产类型**: 支持 app_management、runtime_center 等多种资产类型

### 3. 智能交互引擎

- **多轮对话状态管理**: 维护对话上下文和任务状态
- **意图识别**: 准确识别用户意图并选择合适工具
- **参数澄清**: 当缺少必要参数时主动向用户询问
- **工具调用策略优化**: 根据任务类型选择最优执行路径

## 系统架构

```
AgentSystem
├── 工具层 (Tool Layer)
│   ├── 文件操作工具
│   ├── 系统执行工具
│   └── 搜索查询工具
├── 资产层 (Asset Layer)
│   ├── app_management
│   ├── runtime_center
│   └── master_control
├── 交互层 (Interaction Layer)
│   ├── 意图识别
│   ├── 参数管理
│   └── 状态维护
└── 策略层 (Strategy Layer)
    ├── 工具选择策略
    ├── 脚本优先策略
    └── 收敛控制策略
```

## 应用场景

1. **应用开发**: 创建和管理 draft app，支持完整的开发生命周期
2. **文件操作**: 自动化文件处理流程，支持批量操作

4. **知识问答**: 技术支持和信息查询，提供专业解答
5. **代码辅助**: 代码生成、审查和优化
6. **数据分析**: 数据处理、转换和可视化
