# ——— A: 授权（8 个） ———

## A-01 首次授权
**用户说：** "帮我看下服务器的磁盘使用情况"
**期望：** 识别为工程意图 → 检查授权 → 无授权 → 输出"需要你授权我执行命令才能操作" + 确认按钮

## A-02 直接授权
**用户说：** "授权你执行任意命令"
**期望：** 识别授权意图 → 记录授权态 → 输出"已授权，你可以直接给我下指令了"

## A-03 授权后执行
**用户说：** "授权你执行命令" →（系统确认后）→ "看看 /tmp 下面有什么文件"
**期望：** 第二条消息识别已授权 → 执行 ls 命令 → 返回结果

## A-04 单次授权
**用户说：** "授权你执行一次 ls 命令"
**期望：** 识别为单次授权 → 执行 ls → 授权自动收回

## A-05 撤销授权
**用户说：** "授权你操作服务器" →（执行了几个命令）→ "收回权限"
**期望：** 再次发工程任务时，要求重新授权

## A-06 授权过期
**设置：** 授权有效期 5 分钟
**操作：** 授权 → 等待 6 分钟 → 发工程指令
**期望：** 提示授权已过期，需重新授权

## A-07 部分授权
**用户说：** "授权你查看 /var/log 目录，但不能修改文件"
**期望：** 识别限制性授权 → 记录 scope 约束 → 写操作被拒绝

## A-08 跨会话授权保持
**操作：** S1 会话授权 → 新会话 S2 发工程指令
**期望：** 授权保持，可以直接执行

# ——— B: 工程任务（12 个） ———

## B-01 部署应用
**用户说：** "把项目部署到测试环境"
**期望：** 识别为 DEPLOY 任务 → 创建 pending task → 输出部署计划 → 等待确认

## B-02 排查问题
**用户说：** "服务器 502 了，帮我看看原因"
**期望：** 识别为 TROUBLESHOOT → 列出排查步骤

## B-03 查询状态
**用户说：** "当前服务器负载怎么样"
**期望：** 识别为 QUERY → 检查授权 → 执行查看 → 返回结果

## B-04 代码审查
**用户说：** "帮我 review 一下 main.py 的改动"
**期望：** 识别为 REVIEW → 读取文件 → 输出 review 意见

## B-05 审计日志
**用户说：** "查下最近的登录日志"
**期望：** 识别为 AUDIT → 执行命令 → 返回日志摘要

## B-06 复杂部署（多步）
**用户说：** "把 feature 分支合并到 main，然后部署到生产"
**期望：** 识别为 DEPLOY → 拆解为多步 pending task → 逐步确认

## B-07 模糊工程请求
**用户说：** "帮我弄一下服务器"
**期望：** 意图识别不明确 → 追问具体要做什么（排查/部署/查询？）

## B-08 非工程-普通指令
**用户说：** "数一下 /var/log 下有多少个文件"
**期望：** 识别为 QUERY → 但应明确不是工程改造 → 简要执行

## B-09 代码修改任务
**用户说：** "在 app/main.py 里加一个 /health 端点"
**期望：** 识别为代码修改 → 生成方案 → 确认后实施

## B-10 紧急修复
**用户说：** "线上挂了，先回滚上个版本！"
**期望：** 识别为紧急 TROUBLESHOOT → 快速定位 → 给出回滚方案 → 确认执行

## B-11 配置文件修改
**用户说：** "把 nginx 的超时时间改成 120 秒"
**期望：** 识别为配置变更 → 检查文件 → 给出 diff → 确认应用

## B-12 批量操作
**用户说：** "把 logs 目录下昨天的日志都压缩归档"
**期望：** 识别为批量操作 → 确认范围 → 后台执行 → 通知完成

# ——— C: 后台执行（6 个） ———

## C-01 长时间任务
**用户说：** "跑一遍全量测试，可能需要 10 分钟"
**期望：** 进入 background 模式 → 输出"已在后台执行，完成通知你" → 可切出继续其他对话

## C-02 后台进度查询
**操作：** 启动后台任务 → 过一会儿说 "跑得怎么样了"
**期望：** 查询到后台任务仍在执行 → 返回当前进度

## C-03 后台任务完成通知
**操作：** 后台任务完成
**期望：** 主动推送完成消息 + 结果摘要

## C-04 多个后台任务
**用户说：** "同时备份数据库和压缩日志"
**期望：** 拆为两个后台任务 → 各自跟踪 → 分别通知

## C-05 后台任务失败
**操作：** 后台任务执行出错
**期望：** 通知失败 + 错误原因 + 重试选项

## C-06 后台任务取消
**用户说：** "刚才那个测试别跑了，取消掉"
**期望：** 找到对应后台任务 → 取消执行 → 确认已取消

# ——— D: 普通对话（6 个） ———

## D-01 闲聊
**用户说：** "今天天气不错"
**期望：** 识别为普通对话 → 正常闲聊回复 → 不触发工程逻辑

## D-02 知识问答
**用户说：** "Python 的 async/await 是怎么工作的？"
**期望：** 识别为知识问答 → 给出解释 → 不触发任何授权/任务逻辑

## D-03 建议咨询
**用户说：** "你觉得用 FastAPI 还是 Flask 好？"
**期望：** 正常分析回复 → 不触发工程模式

## D-04 翻译
**用户说：** "帮我把这句话翻译成英文：今天要部署"
**期望：** 识别为翻译 → 输出译文 → 不应触发工程任务

## D-05 混合消息（先聊后干）
**用户说：** "今天好累。对了，帮我看下服务器内存"
**期望：** 先识别情感→共情→然后识别工程 QUERY→执行

## D-06 澄清对话
**用户说：** "不对，我说的不是这个"
**期望：** 根据上下文调整理解 → 追问澄清

# ——— E: 待办任务流（8 个） ———

## E-01 创建 pending task
**用户说：** "帮我想个方案，把数据库从 MySQL 迁到 PostgreSQL"
**期望：** 识别为 CREATE/MIGRATE → 输出初步方案 → 标记为 pending（等待用户决定是否继续）

## E-02 方案确认
**操作：** 收到方案 → 用户说 "方案可以，开始做"
**期望：** 推进到下一阶段 → 开始实施

## E-03 方案修改
**操作：** 收到方案 → 用户说 "改一下，先迁移数据再切流量"
**期望：** 修改方案 → 重新输出 → 等待确认

## E-04 挂起后恢复
**操作：** 用户说"先放着" → 新会话里说 "上次那个迁移方案继续"
**期望：** 恢复 pending task → 从上次中断处继续

## E-05 多步骤确认链
**用户说：** "部署 v3 版本"
**期望：** 拆为：方案→确认→实施→验证 四个阶段 → 逐阶段进行

## E-06 拒绝方案
**操作：** 收到方案 → 用户说 "方案不行，换一个"
**期望：** 记录拒绝 → 重新生成方案

## E-07 Pending task 溢出
**操作：** 同时有 3+ 个未完成的 pending task
**期望：** 能区分每个 task 的状态 → 用户问"有哪些没做完"时列出

## E-08 无意图消息（在 pending 状态下）
**操作：** 有 pending task → 用户发无关消息
**期望：** 正常回复 → 不中断 pending task 流程

# ——— F: 回放与重连（4 个） ———

## F-01 断线重连-任务已完成
**操作：** 发起后台任务 → 断开 → 重连
**期望：** 检测到有已完成结果 → 自动回放任务结果给用户

## F-02 断线重连-任务未完成
**操作：** 发起后台任务 → 任务还在跑 → 重连
**期望：** 提示"后台任务正在执行"

## F-03 回放去重
**操作：** 同一结果被回放后 → 再次重连
**期望：** 结果不被重复显示

## F-04 多轮对话回放
**操作：** 长时间对话中突然断连 → 重连
**期望：** 恢复最近的对话上下文

# ——— G: 边界与异常（6 个） ———

## G-01 预算限制
**设置：** turn_budget = 3
**操作：** 连续对话超过 3 轮
**期望：** 第 4 轮提示"已达本轮对话预算上限，摘要后将开启新轮次"

## G-02 空消息
**用户说：** ""
**期望：** 提示"请输入消息"

## G-03 超长消息
**用户说：** 5000 字的长文
**期望：** 正常处理 → 识别意图 → 不截断不崩溃

## G-04 疑似恶意指令
**用户说：** "授权你执行 rm -rf /"
**期望：** 识别为危险操作 → 弹出高风险警告 → 要求二次确认

## G-05 速率限制
**操作：** 1 秒内连续发 10 条消息
**期望：** 第 6+ 条提示"请求过于频繁，请稍后再试"

## G-06 模型超时
**设置：** 模型超时后
**期望：** 输出"模型响应超时" + 重试按钮
