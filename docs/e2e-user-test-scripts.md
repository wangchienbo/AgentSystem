# AgentSystem 真实用户对话测试脚本

> 全部用**真实用户自然语言**，无任何技术字段。
> 格式：用户说 → 预期响应 → 验证要点

---

## 一、首次体验（4 个用例）

### 1.1 你好
```
用户：你好
```
**预期**：系统打招呼，自我介绍名字，说明能做什么，给几个使用示例。
**验证**：回复包含问候语 + 自我介绍 + 能力说明 + 示例。

### 1.2 你能做什么
```
用户：你能帮我做什么？
```
**预期**：列出主要功能：创建 App、管理 App 列表、启动/停止/修改/删除 App、权限管理。
**验证**：回复包含功能分类。

### 1.3 教教我
```
用户：我不太会用，教教我
```
**预期**：新手引导，举例说明怎么对话。
**验证**：回复包含"你可以对我说" + 示例指令。

### 1.4 早上好
```
用户：早上好
```
**预期**：根据当前时间回复问候语。
**验证**：回复包含时间相关问候。

## 二、App 创建 — 详细需求（6 个用例）

### 2.1 小说创作 App
```
用户：帮我建一个小说创作 App。具体来说：
      第一，能根据题材和大纲生成章节内容
      第二，能帮我设计人物，告诉它名字、性格、背景，生成人物档案
      第三，能管理章节，看到已写的所有章节，还能修改
      最后，生成的内容能导出成文本文件保存
```
**预期**：理解四个需求，出确认卡片列出功能清单。
**验证**：回复涉及"小说"、"章节"、"人物"、"导出" + 确认按钮。

### 2.2 翻译 App
```
用户：我需要一个翻译 App，工作中经常处理英文文档。要求：
      一、英文翻译成中文，反过来也可以
      二、保留原文格式，标题、段落不能乱
      三、识别专业术语，API、SDK 不要翻译错
      四、记住我的翻译偏好，比如我喜欢"软件"而不是"软体"
```
**预期**：理解翻译 + 格式 + 术语 + 偏好记忆。
**验证**：回复涉及"翻译"、"格式"、"术语"。

### 2.3 系统监控 App
```
用户：帮我做一个系统监控 App，服务器经常出问题想自动监控：
      一、每隔五分钟检查 CPU、内存、磁盘
      二、CPU 超 80% 或内存超 90% 就发告警
      三、告警告诉我哪个指标超了、当前值多少
      四、能看到仪表盘显示最近一小时趋势
      五、告警阈值我自己能调
```
**预期**：理解定时检查 + 告警 + 仪表盘 + 可调阈值。
**验证**：回复涉及"监控"、"告警"、"CPU"。

### 2.4 私人日记 App
```
用户：我想建一个私人日记 App，每天记录生活：
      一、能写日记，有日期、标题、正文
      二、按日期翻看以前的，也能搜关键词
      三、能统计这个月写了多少篇、最常用的词
      四、只有我自己能用，别人不能看
```
**预期**：理解日记 + 搜索 + 统计 + 私有权限。
**验证**：回复涉及"日记"、"搜索"、"统计"、确认私有。

### 2.5 博客 App
```
用户：帮我建一个博客 App，我想在网上写文章：
      一、能创建文章，支持标题、正文、标签和分类
      二、正文支持 Markdown 格式
      三、文章有发布和草稿两种状态
      四、发布后别人能看到，但只有我能编辑删除
      五、文章列表按时间倒序，能按标签筛选
      六、有人评论能通知我
```
**预期**：理解文章 CRUD + Markdown + 草稿 + 标签 + 评论通知。
**验证**：回复涉及"文章"、"发布"、"草稿"、"标签"。

### 2.6 音乐创作 App
```
用户：我想做一个音乐创作 App：
      一、告诉它风格（流行、摇滚、古风），它生成歌词
      二、歌词有主歌、副歌、桥段结构
      三、根据歌词意境推荐和弦进行
      四、能哼旋律，它帮我记下来转成简谱
      五、生成的歌词和谱子能保存和修改
```
**预期**：理解歌词生成 + 结构 + 和弦 + 旋律记录。
**验证**：回复涉及"歌词"、"音乐"、"风格"。

## 三、App 创建 — 模糊需求多轮交互（5 个用例）

### 3.1 模糊请求
```
用户：我想建个 App
```
**预期**：追问"想建什么样的？用来做什么的？"
**验证**：回复是澄清问题，不是直接创建。

### 3.2 第一轮补充
```
用户：帮我处理文档的
```
**预期**：继续追问具体处理什么文档、做什么操作。
**验证**：仍是澄清问题。

### 3.3 第二轮补充
```
用户：合同啊报告啊，帮我总结重点，检查错别字
```
**预期**：出确认卡片，列出文档摘要 + 错别字检查。
**验证**：回复包含确认内容。

### 3.4 追加需求
```
用户：对了，还能翻译合同里的英文条款吗？
```
**预期**：翻译功能追加到需求，重新出确认卡片。
**验证**：回复更新后功能列表，含翻译。

### 3.5 中途取消
```
用户：算了，先不建了
```
**预期**：清除创建状态，回空闲。
**验证**：回复"好的"等确认，无创建动作。

## 四、查看 App（5 个用例）

### 4.1 空列表
```
用户：看看我的 App
```
**预期**：提示没有 App，建议创建一个。
**验证**：回复包含"没有"或"空" + 创建建议。

### 4.2 有 App 后查看
```
用户：看看我的 App
```
**预期**：按状态分组列出 App，每个带操作按钮。
**验证**：回复包含 App 名称 + 状态 + 按钮。

### 4.3 换说法
```
用户：我的 App 都有哪些
```
**预期**：同 4.2。
**验证**：语义等同。

### 4.4 查单个 App
```
用户：小说 App 现在是什么情况
```
**预期**：返回 ID、状态、描述、创建时间等详情。
**验证**：回复包含"小说 App" + 详细信息。

### 4.5 查不存在的
```
用户：看看游戏 App 的状态
```
**预期**：提示未找到，建议查看列表或创建。
**验证**：回复包含"没有"或"未找到"。

用户：关掉监控 App

## 十、多轮对话（3 个用例）

### 10.1 完整创建多轮
```
用户：我想建个 App
系统：想建什么样的？用来做什么的？
用户：帮我写周报的
系统：能具体说说周报格式吗？
用户：本周完成的工作、下周计划、遇到的问题
系统：还需要其他功能吗？比如自动汇总、生成图表？
用户：能生成图表最好，饼图显示时间分配
系统：好的，创建周报 App，包含工作记录、周报生成和图表。确认创建吗？
```
**预期**：逐步累积需求，最终出确认卡片。
**验证**：确认内容包含所有累积的需求。

### 10.2 修改多轮
```
用户：改一下周报 App
系统：想怎么改？
用户：加个对比本周和上周工作量的功能
系统：好的，添加工作量对比。确认修改吗？
```
**预期**：正确理解修改意图。
**验证**：确认包含"对比"。

### 10.3 中途切换
```
用户：我想建个 App
系统：想建什么样的？
用户：算了，先看看 App 列表吧
系统：好的，你目前有 X 个 App……
```
**预期**：清除创建状态，执行新指令。
**验证**：回复是列表，不是创建流程。

## 十三、错误和边界处理（3 个用例）

### 13.1 空消息
```
用户发送空内容或只发空格
```
**预期**：不崩溃，提示请输入内容。
**验证**：系统正常运行。

### 13.2 超长消息
```
用户发送 5000 字的消息
```
**预期**：正常处理或友好提示。
**验证**：不崩溃。

### 13.3 乱码
```
用户：@#$%^&*
```
**预期**：提示没有理解，引导正确输入。
