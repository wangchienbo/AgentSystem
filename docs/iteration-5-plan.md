# Iteration 5 验证计划

## 目标
验证北星目标 v1 中尚未充分验证的场景：
1. 重启后 App 和关键状态可以恢复到可用状态
2. LLM / orchestrator / runtime 某一层不可用时，系统能降级而不崩
3. 失败时能定位问题，而不是落到不可解释状态

## 场景设计

### 场景 1: 持久化恢复验证
- 创建 App → 记录状态 → 模拟重启 → 验证状态恢复
- 验证点：
  - App 列表在重启后仍然存在
  - App 状态（启动/停止）能够恢复
  - 上下文历史不丢失

### 场景 2: 降级容错验证
- 模拟 LLM 不可用 → 系统应降级到规则匹配
- 模拟 orchestrator 不可用 → 系统应返回友好错误
- 模拟 runtime 不可用 → 系统应优雅降级

### 场景 3: 错误可解释性验证
- 触发各类错误 → 验证错误信息可定位问题
- 验证日志与错误信息的一致性
- 验证 observability 能追踪完整调用链
